`#바이브코딩` `#CursorAI` `#프롬프트엔지니어링` `#비개발자코딩`

검색 품질을 높이기 위해서는 사용자의 행동 데이터와 피드백을 분석, 문서 적합성을 점검하는 등 다양한 운영 업무가 필요합니다.
이를 위해 개발자와 수시로 소통하고 협업하지만 직접 코드로 구현 작업을 할 일은 없었습니다.
- 따라해본 적은 있지만 기본 설정부터 막히는 부분이 많아 쉽게 지침

AI 개발 도구를 활용하니 아이디어에 집중하여 프롬프트만 요령있게 작성하면 원하는 것을 얻을 수 있었습니다.
덕분에 머릿속 생각을 구현하는 데 필요한 비용이 이전에 비해 낮아지고 도메인 지식을 활용하여 시도할 수 있는 일의 범위가 자연스럽게 확장되어 업무를 바라보는 시각도 달라졌습니다.
이런 변화로 인해 업무에서 어떤 시도를 해서 무엇을 얻었는지, 프롬프트 작성 노하우를 담아 실제 활용에 도움이 되도록 공유하고자 합니다.
## 🔍 바이브 코딩이란
> 👨‍💻 **바이브 코딩(Vibe Coding)**
> 코드를 직접 작성하지 않아도** 자연어 입력**이나 **이미지 업로드만**으로 프로그램을 빠르게 만들 수 있는 <span class="text-red">대화형 개발 방식</span>

바이브 코딩이 가능해진 배경에는 <span class="text-blue">**생성형 AI**</span>의 급격한 발전이 있습니다.
특히 **대규모 언어 모델(LLM)**이 고도화되어 문장 전체의 흐름과 의도를 파악해 복잡한 요구사항도 논리적으로 처리합니다.
- 코드 생성에 특화된 **코딩 LLM**이 등장하면서 단순한 코드 생성에서 벗어나 전체 프로젝트 구조를 설계하고 기존 코드와의 연결까지 고려하는 수준으로 발전
- 멀티 모달 입력 기술이 더해져 **자연어뿐 아니라 이미지나 설계 자료**를 통해 원하는 기능을 구현할 수 있게 됌
## 👋 바이브 코딩 맛보기
`예시: Cursor AI`
커서 AI는 생성형 AI를 결합한 통합 개발 환경으로 요구사항을 담은 문장을 입력 시 필요한 동작•디자인•데이터 연결등이 반영된 코드를 작성해줍니다.
- 기존 프로젝트가 있다면 알맞은 위치에 새로 생성한 코드를 연결, 변수 이름 정리•버그 수정•코드 다듬기 같은 작업도 실시간으로 제안

## 🧑‍💻 검색어 이상치 탐지 프로그램 만들기
> **업무에 바이브 코딩을 활용한 실전 사례**

검색 운영 업무에서는 검색어의 지표를 살피는 일이 중요
- `클릭률(CTR)`, `전환율(CVR)` 같은 지표를 살펴보다 → 빠르게 대응

⚠️ **검색어는 수천, 수만개이며 데이터는 시시각각 변하기 때문에 사람의 눈으로 모든 변화를 확인하기에는 한계가 존재**
### 검색어 이상치 탐지 프로그램
- 커서 AI를 이용해 <u>검색어 이상치 탐지 프로그램</u>을 만들어서 처리!
- 주요 지표의 시계열 데이터를 분석해 평소와 다른 변화가 감지되는 검색어만 선별하여 노출

1️⃣** 분석할 키워드 필터링**
**2️⃣ 키워드를 검색 빈도로 분류하고 샘플링**
**3️⃣ 키워드별 상세 데이터를 가져옴**
- 선택된 키워드의 클릭률, 전환율, 노출 횟수등의 지표를 빅쿼리에서 자동으로 추출해 분석 준비
**4️⃣ 이상 징후를 찾아낸다.**
- 시계열 데이터 분석을 통해 지표에서 급격한 변화를 자동으로 감지하여 이상치를 찾아냄
**5️⃣ 결과를 보기 쉬운 그래프 형식으로 만듬**
**6️⃣ 파일로 결과를 저장**

이런 전체적인 흐름을 구상하고 기능을 하나씩 쌓아가는 과정은** AI와 협업**하듯 진행됐습니다.
- 필요한 기능이 생각나면 자연어로 설명하고 구현은 커서 AI가 맡아서 처리
- 중간중간에 AI가 되묻거나 제안을 하면서 코딩 전문 조수를 곁에 둔 것 같은 느낌
- 추가 요청을 하며 원하는 모습이 나올 때까지 반복
## 🤖 인터렉션 프로토타입으로 요구사항 전하기
> **바이브 코딩으로 인터렉션 프로토타입을 만들어 팀원과의 의사소통 효율을 높인 사례**

보통은 **개발 요구사항을 준비하고 전달하는 단계**에서 개발자가 요구사항을 명확하게 이해할 수 있도록 텍스트 중심의 문서를 상세히 작성하고 이해를 돕기위해 정적인 화면을 그려넣기도 합니다.
문서가 준비되면 서로의 이해도를 맞추기 위한 <u>싱크 회의</u>를 진행합니다.

⚠️ **이러한 방식은 요구사항 문서 준비에 상당한 시간을 필요로 하며 문서를 읽고 팀원 간 이해를 확인하고 조율하는데 높은 커뮤니케이션 비용이 발생**

### 인터렉션 프로토타입
개발자가 직접 눈으로 보고 직관적으로 이해할 수 있는 인터렉션 프로토타입으로 요구사항을 전달
바이브 코딩으로 만든 프로토타입으로 요구사항을 전달하니 이전보다 빠르고 정확하게 개발자와 커뮤니케이션을 할 수 있었습니다.
- 머릿속에 그리는 화면과 흐름을 그대로 공유 가능
- 이해의 차이로 생기는 혼선을 줄이고 화면을 보며 구체적인 피드백을 주고받아 싱크 회의 시간 단축

### 🫡 바이브코딩을 통해
무엇을 그리고 싶은지 자연어로 명확하게 설명하면 AI가 동작 가능한 형태로 빠르게 구현을 해주어 팀원들과 효율적으로 논의하고 발전시킬 수 있었습니다.
- 분석•개발•운영 사이의 간극을 줄이고 협업 속도를 크게 높일 수 있었습니다.

## 🤔 이 챕터를 읽고나서,,, 
**1️⃣ 프롬프트가 진짜 중요하다!!**
단순히 '코드 짜줘'라고 요청하는 것을 넘어 비즈니스 로직이나 데이터 흐름, 예외 처리를 얼마나 논리적으로 AI에게 설명하느냐에 따라 결과물의 퀄리티가 확 달라진다는 걸 다시 느꼈습니다. 머리로는 알고 있었지만 이 챕터를 읽고 나니 프롬프트 작성법을 더 제대로 연습해야겠다는 생각이 들었다.
또, 물론 언어나 프레임워크의 문법을 아는 것도 여전히 중요하겠지만 이제는 요구사항을 명확하게 정의하고 이를 쪼개서 생각하는 `컴퓨팅 사고력`이 정말 중요한 시대가 된 것 같다..!

**2️⃣ 커뮤니케이션 패러다임**
실제로 네이버 부스트캠프 그룹 프로젝트를 할 때, 와이어프레임 같은 시각 자료를 보면서 팀원들과 논의하니 훨씬 집중도 잘 되고 요구사항 이해가 쉬웠고 이를 바탕으로 API 스펙이나 DB 스키마를 설계하는 데도 많은 도움이 되었었는데 바이브 코딩으로 빠르게 프로토타입을 딸깍 만들어서 소통한다면 시간도 엄청 단축될 것 같다. 내 아이디어를 바로 시각화해서 팀원들을 설득할 수 있는 도구가 생겼다는 생각이 들었다.

**3️⃣ 도메인 지식은 여전히 중요하다.**
이 챕터의 검색어 이상치 탐지 사례를 보면서 느낀 건 코드는 AI가 다 짜주더라도 '수많은 지표(CTR, CVR 등) 중 뭘 볼 건지', '무엇을 이상치로 정의할 건지' 기준을 세우는 건 결국 내가 가진 **도메인 지식**이라는 점이라는 것...!
즉, AI는 내 요구사항을 처리해 주는 동료일 뿐 결국 문제 해결의 방향키를 쥐고 있는 건 AI를 사용하는 우리들(개발자)이라는 걸 잊지 말아야겠다..!
