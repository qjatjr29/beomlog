## 🔍 운영체제(Operating System, OS)
<img src="https://www.notion.so/image/https%3A%2F%2Fprod-files-secure.s3.us-west-2.amazonaws.com%2Fce63075d-cee3-44a7-8d31-efcf5784b0e7%2F509ac83f-ff68-4e4a-8380-8529995c3eb6%2Fimage.png?table=block&id=31ad67b9-9e80-8087-8f53-c60a567d6d74&cache=v2" alt="image" width="632" height="508" loading="lazy" />

컴퓨터 하드웨어 바로 위에 설치되어 사용자 및 다른 모든 소프트웨어와 하드웨어를 연결하는 **소프트웨어 계층**
- 컴퓨터 하드웨어(CPU, 메모리, 입출력 장치 등)를 관리
- 응용 프로그램이 원활하게 실행되도록 자원을 할당
- 사용자와 컴퓨터가 상호작용 할 수 있도록 사용자 인터페이스 제공

**📌 좁은 의미 vs 넓은 의미**
- **좁은 의미 (커널, Kernel)**: 운영체제의 핵심 부분으로 컴퓨터가 부팅된 후 메모리에 상주하며 하드웨어와 직접 통신하고 시스템의 모든 기능을 수행합니다.
- **넓은 의미**: 커널뿐 아니라 주변 시스템 유틸리티(파일 탐색기, 작업 관리자 등)를 모두 포함한 개념입니다.

**운영체제는 응용 프로그램이나 사용자에게 컴퓨터 자원을 사용할 수 있는 인터페이스를 제공하고 그 결과를 돌려준다.**
- ⚠️ 응용 프로그램이나 사용자에게 컴퓨터 자원을 숨기고 정해진 방법으로만 컴퓨터 자원을 사용할 수 있도록 제한
- 운영체제를 거치지 않고서는 하드웨어에 접근할 수 없다 ❌

**운영체제는 응용 프로그램에게 컴퓨터 자원을 적절히 나누어준다.**

## 🎯 운영체제의 목적과 역할
### 🎯 목적
1️⃣ **컴퓨터 시스템 자원을 효율적으로 관리**
- 프로그램들에게 짧은 시간씩 CPU를 번갈아 할당
- 프로그램들에 메모리 공간을 적절히 분배
- I/O 장치(디스크, 네트워크)를 효율적으로 제어
- 형평성 있는 자원 분배로 주어진 하드웨어에서 최대 성능을 끌어내고 프로세스, 파일, 메시지 등을 관리

2️⃣ **컴퓨터 시스템을 편리하게 사용할 수 있는 환경 제공**
- 동시에 프로그램들이 동작하는 것처럼 보이도록 해줌
- 하드웨어를 직접 다루는 복잡한 부분을 운영체제가 대행

3️⃣ **보호와 안정성**
- 사용자 간 자원 침범을 막고 시스템 전체 신뢰성을 유지

### 👷‍♂️ 역할

| 역할 | 설명 | 핵심 기능 |
| ---- | ---- | ---- |
| 자원 관리 | 여러 자원을 프로그램에 적절히 분배·회수하며 원활한 작업 환경 제공 | CPU 스케줄링, 메모리 할당, I/O 큐잉 |
| 자원 보호 | 악의적/오류로 인한 침범 방지(예: 버퍼 오버플로우로 인한 메모리 훼손) | 유저/커널 모드 분리, 접근 제어(ACL) |
| 하드웨어 인터페이스 | 복잡한 하드웨어(CPU, 메모리, 키보드 등)를 표준화된 API로 추상화 | 시스템 콜, 드라이버 통해 일관된 접근 지원 |
| 사용자 인터페이스 | CLI(명령어), GUI(그래픽)로 사용자와 소통 | 셸(Bash), 데스크톱 환경(Windows Explorer) 제공(보강) |

> 📖 **ACL (Access Control List, 접근 제어 목록)**
> 파일, 디렉토리, 프로세스 등 시스템 자원에 대해 **누가(User/Group),** **어떤 권한(Read/Write/Execute)**으로 접근할 수 있는지 정의한 목록

## 🧩 운영체제의 분류

### **동시 작업 가능 여부**
**단일 작업 (Single Tasking)**: 한 번에 하나의 작업만 처리 (예: MS-DOS)
**다중 작업 (Multi Tasking) ✅**: 동시에 두 개 이상의 작업 처리 (예: UNIX, Windows)

### **사용자의 수**
**단일 사용자 (Single User)**: 한 명의 사용자만 접근 가능 (예: MS Windows)
**다중 사용자 (Multi User) ✅**: 여러 사용자가 동시에 접속하여 서버 자원 공유 (예: UNIX, Linux)

### **처리 방식**
**일괄 처리 (Batch Processing)**: 작업 요청을 모아 한꺼번에 순차적으로 처리. 응답시간이 길다. (초기 OS)
**시분할 (Time Sharing) ✅**: CPU 시간을 잘게 쪼개어 여러 작업에 할당. 사용자에게 빠른 응답성(Interactive) 제공. (현대 범용 OS)
**실시간 (Realtime)**: 정해진 시간(Deadline) 안에 반드시 작업 종료가 보장되어야 하는 시스템.
- **경성 실시간 (Hard Realtime) 🔥**: 데드라인을 어기면 치명적인 결과 발생. (예: 무기 제어, 원자로 제어, 항공기 자율주행)
- **연성 실시간 (Soft Realtime) ☁️**: 데드라인을 맞추는 것이 목표지만 가끔 어겨도 품질 저하만 발생할 뿐 치명적이진 않음. (예: 동영상 스트리밍, 온라인 게임)

## 🔐 권한 모드** (Dual Mode)**
응용 프로그램이 하드웨어에 무차별적으로 접근하여 시스템을 망가뜨리는 것을 막기 위해 CPU는 `이중 모드(Dual Mode)`를 지원
- **유저 모드 (User Mode)**: 일반 응용 프로그램이 실행되는 모드. <u>하드웨어 직접 접근이 불가하며 제한된 명령만 수행.</u>
- **커널 모드 (Kernel Mode)**: OS 커널 코드가 실행되는 모드. <u>모든 하드웨어 및 특권 명령어(Privileged Instructions)에 접근 가능.</u>

### 유저 모드에서 커널 모드로 전환
1️⃣ **Exception 발생**
프로그램에서 0으로 나누기 같은 오류가 났을 때 발생하기도 하고 사용자, 프로그램이 OS 서비스를 요청하기 위해 `시스템 콜`을 통해서 의도적으로 예외(trap)을 발생시키는 경우가 있다.
- 유저 모드에서 하드웨어 자원이 필요할 때 <span class="text-blue">**시스템 콜(System Call)**</span>을 발생시키면 CPU가 커널 모드로 전환(Trap)되어 작업을 수행하고 다시 유저 모드로 복귀

2️⃣ **인터럽트** 
컴퓨터 시스템에서 발생하는 이벤트 중 하나로 CPU가 현재 실행 중인 작업을 중단하고 해당 이벤트를 처리하기 위해 운영 체제 내부의 인터럽트 핸들러 함수를 실행.
일반적으로 하드웨어에서 발생하는 이벤트(하드웨어 인터럽트)
- I/O 작업 완료(키보드 입력, 디스크 읽기 등)
- 타이머 인터럽트 (시분할 스케줄링. 타임 슬라이스가 만료된 경우)

> 💡 <span class="text-blue">**인터럽트**</span> 같은 경우는 다음 글에서 더 자세히 알아보고자 합니다.

> 운영체제의 핵심 부분으로 하드웨어와 직접 통신하며 시스템의 모든 기능을 수행


🔖 **핵심 기능**

| **I/O 관리** | 장치 간 데이터 전송 및 인터럽트 처리 | 인터럽트 핸들러, DMA(Direct Memory Access), I/O 스케줄러, 블록 레이어 |
| **프로세스간 통신(IPC)** | 프로세스 간 데이터 교환 및 동기화 | 파이프, 메시지 큐, 세마포어/뮤텍스, 공유 메모리, 시그널 |

### 시스템 콜
커널은 사용자나 응용 프로그램으로부터 자원을 보호하기 위해 자원에 직접 접근하는 것을 차단한다.
따라서, 하드웨어 같은 시스템 자원을 이용하기 위해서는 **`시스템 콜`**** 인터페이스**를 이용해 접근해야 한다.

> **예시) 응용 프로그램이 데이터를 하드디스크에 저장하려고 하는 상황**

시스템 콜을 통해서 응용 프로그램은 직접 하드디스크에 데이터를 저장하지 않고 커널이 제공해주는 `write()` 함수를 이용해서 데이터를 저장해달라고 요청
- 응용 프로그램은 데이터가 하드디스크의 어느 위치에 어떤 방식으로 저장되는지 알 수 없고 자신이 저장한 데이터를 읽고 싶다면 `read()`함수로 시스템 콜을 이용해 가져온다.
커널이 데이터를 가져오거나 저장하는 것을 책임지기 때문에 자원을 관리하기 쉽다.

### 인터페이스
사용자와 응용 프로그램이 컴퓨터 하드웨어(CPU, 메모리 등)를 직접 다루는 대신 OS의 기능과 서비스를 안전하고 쉽게 사용하도록 해주는 상호작용 방식.
- **CLI(명령줄 인터페이스)**, **GUI(그래픽 인터페이스)**, **시스템 호출(System Call)**


| **인터페이스 ** | **설명** | **예시** |
| ---- | ---- | ---- |
| **CLI** | 마우스나 터치스크린을 사용하여 아이콘, 창 등 그래픽 요소를 통해 운영체제를 조작 | Windows, macOS, Android |
| **GUI** | 텍스트 기반의 명령어(터미널/명령 프롬프트)를 직접 입력하여 운영체제에 명령 | Bash, PowerShell |
| **System Call** | 응용 프로그램이 커널의 자원을 안전하게 사용하기 위해 요청하는 API(응용 프로그램 인터페이스) |  |
